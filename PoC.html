Systemspecificering för IoT-baserad batteriövervakning för telemaster

1. Inledning

Denna Proof of Concept (PoC) demonstrerar ett IoT-system för temperaturövervakning. Systemet mäter temperatur med en Arduino-baserad sensor som skickar datan till en backend via HTTP och visualiserar värden i en webbläsare.

Målet är att visa hur data från en fysisk sensor kan samlas in, lagras och presenteras i realtid i ett webbaserat gränssnitt.

2. Systemarkitektur

2.1 Översikt

Systemet är uppbyggt enligt följande flöde:

Arduino (sensor) <--> PC-program (via UART) --> HTTP --> Backend (Spring Boot och SQL) <-- Webfrontend

2.2 Mikrokontroller (Arduino)

  • Utrustad med en temperatursensor
  • Väntar på att ta emot en instruktion från PC-programmet via UART.
  • När instruktionen tas emot, läser mikrokontrollern sensordatan och skickar tillbaka temperaturvärdet över UART.

2.3 PC-program

  • Fungerar som en brygga mellan backend och mikrokontroller.
  • Skickar instruktionen till mikrokontrollern.
  • Tar emot temperatursvaret via UART.
  • Skickar värdet vidare till backend via HTTP POST.

2.4 Backend

  • Tar emot data via ett REST-endpoint.
  • Lagrar temperaturvärden i en SQL-databas.
  • Tillhandahåller en frontend som visar de senaste mätningarna.

2.5 Frontend

  • Webbsida byggd i Spring Boot och thymeleaf.
  • Hämtar data från backend och visar temperaturhistorik.

4. Säkerhet

Systemet fokuserar på funktionalitet snarare än säkerhet.
I denna version ingår ingen kryptering eller autentisering mellan mikrokontroller, PC och backend.
All kommunikation sker i klartext via UART och HTTP.
För framtida versioner kan följande säkerhetsförbättringar övervägas:

  • HTTPS: För att skydda datan mellan PC och backend.
  • API-nyckel / autentisering: För att säkerställa att endast auktoriserade klienter kan skicka data.
  • Krypterad UART-kommunikation: För att skydda seriell data från manipulation.
  • Firmware-säkerhet (Secure Boot): För att förhindra att mikrokontrollern kör osignerad kod.
